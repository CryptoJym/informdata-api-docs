Authentication

The InformData Connect API uses JWT (JSON Web Token) authentication. All API requests must include a valid JWT token in the Authorization header.

Authentication Flow

1

Get Token

POST credentials to /api/v1/authtoken

→
2

Receive JWT

Valid for 3 hours

→
3

Make Requests

Include token in headers

→
4

Refresh Token

Before expiration

Get Authentication Token

curl -X POST https://api.informdata.com/api/v1/authtoken \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your_username",
    "password": "your_password"
  }'
const response = await fetch('https://api.informdata.com/api/v1/authtoken', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'your_username',
    password: 'your_password'
  })
});

const { token } = await response.json();
console.log('JWT Token:', token);
import requests

response = requests.post(
    'https://api.informdata.com/api/v1/authtoken',
    json={
        'username': 'your_username',
        'password': 'your_password'
    }
)

token = response.json()['token']
print(f'JWT Token: {token}')

Token Expiration

JWT tokens expire after 3 hours (10,800 seconds). Implement token refresh logic to maintain uninterrupted access.

Criminal Background Checks

Criminal background checks search court records to identify any criminal history. We offer multiple types of criminal searches to meet your screening needs.

Available Criminal Check Types

County Criminal (CRIMINAL)

Search criminal records at the county courthouse level. Most thorough and accurate for specific jurisdictions.

Configuration Parameters:

Parameter Type Values Description
felonyYears integer 1-10 Years to search for felony records
felonyScope string STRICT_SCOPE, EXTENDED_SCOPE, BROAD_SCOPE Geographic scope for felony search
misdemeanorYears integer 1-10 Years to search for misdemeanor records
misdemeanorScope string STRICT_SCOPE, EXTENDED_SCOPE, BROAD_SCOPE Geographic scope for misdemeanor search
trafficLevel string NO_TRAFFIC, TRAFFIC_MISDEMEANORS_ONLY, ALL_TRAFFIC Include traffic violations

Example Request:

{
  "order": {
    "clientOrderId": "CRIM-2024-001",
    "subject": {
      "FirstName": "John",
      "LastName": "Doe",
      "Birthdate": "1990-01-01"
    },
    "product": {
      "category": "CRIMINAL",
      "felonyYears": 7,
      "felonyScope": "STRICT_SCOPE",
      "misdemeanorYears": 7,
      "misdemeanorScope": "STRICT_SCOPE",
      "trafficYears": 7,
      "trafficScope": "STRICT_SCOPE",
      "trafficLevel": "TRAFFIC_MISDEMEANORS_ONLY"
    },
    "jurisdiction": {
      "county": "King",
      "state": "WA",
      "country": "United States"
    }
  }
}

Federal Criminal (CRIMINAL_FEDERAL)

Search federal district court records for federal crimes across all US federal jurisdictions.

Federal searches cover crimes prosecuted at the federal level, including interstate crimes, federal tax violations, and crimes on federal property.

National Criminal Database (NATCRIM)

Instant search across a multi-jurisdictional database. Good for identifying potential records that need county-level verification.

Database searches should be supplemented with county-level searches for complete due diligence.

Understanding Criminal Check Responses

Response Handling

All API responses follow a consistent structure, making it easy to parse and handle results programmatically.

Response Structure

{
  "order": {
    "id": "string",                    // InformData order ID
    "clientOrderId": "string",         // Your reference ID
    "orderDate": "datetime",
    "dueDate": "datetime",
    "status": "COMPLETE",             // Order status
    "substatus": "CLEAR",             // Result status
    "subject": { ... },               // Subject information
    "product": { ... },               // Product configuration
    "jurisdiction": { ... },          // Location details
    "activity": [ ... ],              // Status updates
    "cases": [ ... ],                 // Results (if applicable)
    // Category-specific fields
  }
}

Status Values

Order Status

  • CREATED - Order received
  • IN_PROGRESS - Processing
  • COMPLETE - Finished
  • ON_HOLD - Requires attention
  • CANCELLED - Cancelled

Substatus by Type

Criminal/Civil Checks

  • CLEAR - No records found
  • RECORD - Records found
  • POSSIBLE_RECORD - Needs review

Verifications

  • VERIFIED_SUCCESS - Verified
  • VERIFIED_FAILURE - Mismatch
  • UNABLE_TO_VERIFY - Cannot verify

Webhooks

Receive real-time notifications when order status changes. Webhooks eliminate the need for polling and ensure you're immediately notified of results.

Setting Up Webhooks

POST /api/v1/events/subscriptions
Content-Type: application/json
Authorization: Bearer YOUR_JWT_TOKEN

{
  "url": "https://your-domain.com/webhook",
  "events": [
    "order.created",
    "order.status.changed",
    "order.completed"
  ],
  "secret": "your_webhook_secret"
}

Webhook Events

Event When Triggered Payload
order.created New order created Order ID, initial status
order.status.changed Status update Order ID, old/new status
order.completed Processing complete Full order details with results
order.updated Order modified Order ID, changed fields

Webhook Security

Verify webhook authenticity by validating the signature in the X-InformData-Signature header using your webhook secret.

Error Handling

The API uses standard HTTP status codes and returns detailed error information to help you quickly identify and resolve issues.

Error Response Format

{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error description",
    "details": {
      // Additional context
    }
  }
}

Common Error Codes

Client Errors (4xx)

  • 400 - Bad Request (invalid data)
  • 401 - Unauthorized (invalid/expired token)
  • 403 - Forbidden (insufficient permissions)
  • 404 - Not Found
  • 429 - Rate Limit Exceeded

Server Errors (5xx)

  • 500 - Internal Server Error
  • 503 - Service Unavailable

Best Practices

Authentication

  • Store JWT tokens securely (never in client-side code)
  • Implement token refresh before expiration (3-hour lifecycle)
  • Use HTTPS for all API communications

Request Handling

  • Always include a unique clientOrderId for tracking
  • Validate required fields before sending requests
  • Use appropriate search scopes for criminal checks
  • Include SSN for more accurate criminal searches when available

Response Processing

  • Check both status and substatus fields
  • Handle all possible status combinations
  • Store the InformData orderId for future queries
  • Implement proper error handling for all status codes

Performance Optimization

  • Use webhooks instead of polling for status updates
  • Batch requests when possible
  • Implement exponential backoff for retries
  • Cache authentication tokens until expiration

Compliance & Security

  • Only request necessary screening products
  • Secure all PII data in transit and at rest
  • Implement audit logging for all API calls
  • Follow FCRA guidelines for background checks

API Endpoints Overview

The InformData Connect API provides RESTful endpoints for all operations. All endpoints require authentication via JWT token.

Base URL

https://api.informdata.com

Core Endpoints

Authentication

  • POST /api/v1/authtoken - Get authentication token
  • POST /api/v1/authtoken/refresh - Refresh token

Orders

  • POST /api/v1/orders - Create new order
  • GET /api/v1/orders/{orderId} - Get order details
  • GET /api/v1/orders - List orders
  • PUT /api/v1/orders/{orderId} - Update order
  • DELETE /api/v1/orders/{orderId} - Cancel order

Webhooks

  • POST /api/v1/events/subscriptions - Create webhook
  • GET /api/v1/events/subscriptions - List webhooks
  • DELETE /api/v1/events/subscriptions/{id} - Delete webhook

For complete endpoint documentation with request/response examples, see the detailed endpoints reference.
